### 0.5.7
- Fix `flatStructure` on form blocks.

### 0.5.6
- SchedulerGrid: stop generating `sameAsPrev` / `prevData` flags during manual dialog edits; modification markers now remain input-driven only
- SchedulerGrid: update `SICK` and `HALF_DAY` example legend colors to improve visual separation in light mode

### 0.5.5
- Add new property `openByDefault` for Expansion panel.

### 0.5.4
- SchedulerGrid: fix header generation when first user has empty `schedule` (headers are now resolved from all provided schedules)
- SchedulerGrid: fix cell alignment between users with different day ranges by matching entries using day keys (`date` / `day` + optional `group`) instead of raw array index
- SchedulerGrid: sort merged headers by `date` for mixed user ranges (with fallback sort by `day` when date is not provided)
- Storybook: add regression scenario `Empty First User + Mixed Ranges`
- Storybook: add regression scenario `Mixed Ranges - Sorted Headers`

### 0.5.2
- SchedulerGrid: render day headers from provided schedule order (instead of fixed `1..N`) with backward compatibility for month/year context
- SchedulerGrid: add optional day grouping via `group` and new schema flag `showGroupHeaders`
- SchedulerGrid: support selected/custom day subsets (including non-contiguous and cross-month groups)
- SchedulerGrid: compact columns for weekends without business data to reduce visual noise
- SchedulerGrid: improve modified-day indicator layering so corner marker remains visible over neighboring cells on hover
- Storybook: add `With Selected Range`, `With Grouped Days`, `With Grouped Days - Hidden Groups`, and `Customization - Groups + Modified Data`

### 0.5.0
- SchedulerGrid: add support for `sameAsPrev` + `prevData` to highlight modified days and de-emphasize unchanged days
- SchedulerGrid: add modified-day corner indicator and tooltip section with previous status/note values
- SchedulerGrid: localize status labels by `statusKey` via `useLocale` (legend, tooltip, edit select)
- Storybook: add `With Modified Data` and `With Modified Data - Readonly` scenarios (31-day datasets)
- improvements on JSON resolver with params
